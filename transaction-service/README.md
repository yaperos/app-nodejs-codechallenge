## Features
- **Hexagonal Architecture**: The microservice follows the hexagonal architecture pattern, allowing for clear separation of concerns and easy maintenance.
- **PostgreSQL**: Data persistence is managed through PostgreSQL, providing reliability and scalability.
- **Redis Caching**: Redis is used for caching frequently accessed data, improving performance.
- **Kafka Event Streaming**: Kafka facilitates event streaming for real-time communication between microservices.
- **OpenAPI Documentation**: API documentation is generated using OpenAPI, ensuring clear and comprehensive documentation.
- **NestJS**: The microservice is built with NestJS, a powerful framework for building efficient and scalable server-side applications.
- **Yarn**: Dependency management is handled using Yarn, providing fast and secure package installations.


## Getting Started
To get started with the Transaction Microservice Worker, follow these steps:

1. **Clone the Repository:**
   ```bash
   git clone git@github.com:mssalazarb/app-nodejs-codechallenge.git
   ```

2. **Install Dependencies:**
   ```bash
   yarn install
    ```

3. **Set Up Environment Variables:**
   ```bash
   cp .env.example .env
   ```

4. **Build microservice:**
   ```bash
   npm run build
   ```

5. **Run migrations:**
   ```bash
   npm run migrate
   ```

6. **Run the Microservice Kafka Worker:**
   ```bash
   node dist/worker
   ```

7. **Run the Microservice Server:**
   ```bash
   npm run start:dev
   ```
